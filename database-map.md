# 🗄️ Dabba v4 — Data Sources & Storage

## Data Storage Strategy

Dabba uses a **dual storage** approach with a clear migration path:

1. **CSV files** — Primary data source for the ML pipeline (pandas DataFrames)
2. **SQLite/Postgres (via SQLAlchemy)** — Used for persisting restaurant data, experiment results, drift logs, and orders

The migration path is: SQLite (dev) → Postgres (prod), managed by Alembic.

### CSV → Database Migration

```bash
# One-command full import: CSV → clean → feature engineer → seed to DB
make db-import

# Or manually:
python -m dabba.database.seed --full-import

# Individual seeding (from processed CSVs):
python -m dabba.database.seed
python -m dabba.database.seed --clear  # Clear before seeding
```

The API reads from the database (not CSV) via `load_zomato_from_db()` and `load_delivery_from_db()`, with automatic CSV fallback when the database is empty.

## Database Schema (SQLAlchemy ORM)

### `restaurants`
| Column | Type | Description |
|--------|------|-------------|
| `id` | INTEGER PK | Auto-increment |
| `name` | VARCHAR(255) | Restaurant name (indexed) |
| `rate` | FLOAT | Parsed rating (0-5) |
| `bayesian_rating` | FLOAT | Bayesian-adjusted rating |
| `cost_for_two` | FLOAT | Cost in INR |
| `location` | VARCHAR(255) | Bangalore neighborhood |
| `cuisines` | TEXT | Comma-separated cuisines |
| `votes` | INTEGER | Review count |
| `votes_log` | FLOAT | log1p(votes) |
| `online_order_binary` | INTEGER | 0/1 |
| `book_table_binary` | INTEGER | 0/1 |
| `cuisine_count` | INTEGER | Number of cuisines |
| `avg_sentiment` | FLOAT | VADER compound score |
| `reliability_score` | FLOAT | Composite score (0-1) |
| `latitude` / `longitude` | FLOAT | Restaurant coordinates |
| `created_at` / `updated_at` | DATETIME | Auto-timestamps |

### `orders`
| Column | Type | Description |
|--------|------|-------------|
| `id` | INTEGER PK | Auto-increment |
| `restaurant_id` | INTEGER FK | References restaurants.id |
| `distance_km` | FLOAT | Haversine distance |
| `traffic_level` | INTEGER | 0-3 ordinal |
| `is_festival` | BOOLEAN | Festival flag |
| `delivery_person_age` / `delivery_person_rating` / `vehicle_condition` | FLOAT/INT | Profile fields |
| `predicted_eta` / `actual_eta` | FLOAT | Predicted vs actual delivery time |
| `is_at_risk` / `actual_late` | BOOLEAN | SLA status flags |

### `predictions`
| Column | Type | Description |
|--------|------|-------------|
| `id` | INTEGER PK | Auto-increment |
| `model_name` / `model_version` | VARCHAR | Model identity |
| `input_hash` | VARCHAR(64) | SHA-256 of serialized input (indexed) |
| `input_data` | JSON | Raw input for explainability |
| `output_value` | FLOAT | Model prediction |
| `shap_values` | JSON | SHAP explanations |

### `experiment_results`
| Column | Type | Description |
|--------|------|-------------|
| `id` | INTEGER PK | Auto-increment |
| `task` | VARCHAR(20) | 'rating' or 'eta' |
| `model_name` | VARCHAR(100) | Algorithm name |
| `mae` / `rmse` / `r2` / `train_time_s` | FLOAT | Metrics |
| `mlflow_run_id` | VARCHAR(50) | MLflow link |
| `is_winner` | BOOLEAN | Best model flag |

### `drift_logs`
| Column | Type | Description |
|--------|------|-------------|
| `id` | INTEGER PK | Auto-increment |
| `feature_name` | VARCHAR(100) | Feature that drifted (indexed) |
| `ks_statistic` / `p_value` | FLOAT | KS-test results |
| `threshold` | FLOAT | p-value threshold used |
| `n_reference` / `n_batch` | INTEGER | Sample counts |
| `alerted` | BOOLEAN | Slack notification sent? |
| `detected_at` | DATETIME | When drift was detected |

## Database Access Patterns

### Reading (Repositories)
```python
from dabba.database.repositories import get_all_restaurants, get_restaurant_by_id
from dabba.database.session import get_db_generator

# FastAPI dependency injection
@app.get("/v1/restaurants")
def list_restaurants(db: Session = Depends(get_db_generator)):
    return get_all_restaurants(db, limit=50)
```

### Writing (Seed)
```python
from dabba.database.seed import seed_restaurants, seed_orders, clear_all

# Upsert restaurants by name (existing → update, new → insert)
seed_restaurants(df_zomato, config)

# Insert orders (pure insert, no upsert)
seed_orders(df_delivery, config=config)

# Clear all data (single transaction)
clear_all(config)
```

### DB-Backed Loaders
```python
from dabba.data.loaders import load_zomato, load_delivery

# Try DB first, fall back to CSV
df = load_zomato(use_db=True)

# Force CSV load
df = load_zomato(use_db=False)
```

## Alembic Migrations

The database schema is managed by Alembic:

```bash
# Create a new migration
make db-revision message="add new column"

# Apply migrations
make db-migrate

# Rollback
make db-rollback

# Show history
make db-history
```

Migration files are in `alembic/versions/`. The initial migration creates all 5 tables.

### Docker Startup

The `docker/entrypoint.sh` runs `alembic upgrade head` before starting uvicorn, ensuring the database schema is always up-to-date when the container starts.

## Entity Relationships

```
Restaurant (zomato.csv → restaurants table)
    │
    ├── has many reviews → VADER sentiment analysis
    ├── has many cuisines → one-hot encoded
    ├── has one location → Bangalore area → city_zone
    └── has one rating → predicted by winning rating model
         │
         └── feeds into Reliability Score
              └── feeds into Hybrid Recommender

Delivery Order (deliverytime.csv)
    │
    ├── has one restaurant location
    ├── has one delivery location
    ├── has one delivery partner
    ├── has weather conditions → weather_encoded
    ├── has traffic density → traffic_ordinal
    └── has one time_taken → predicted by winning ETA model
         │
         ├── feeds into Reliability Score
         ├── feeds into SLA early-warning logic
         └── feeds into Drift Detection → Slack alert
              └── logged in drift_logs table

Synthetic User (generated → synthetic_interactions.csv)
    │
    ├── has many restaurant interactions
    └── assigned cuisine/price preference profile
         │
         └── feeds into Matrix Factorization → collaborative scores
              └── feeds into Hybrid Recommender
```
