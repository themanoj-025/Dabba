# 💾 Dabba — Data Map

## Overview

**Dabba does not use a database.** All data is stored as CSV files and loaded into pandas DataFrames for in-memory processing. This is appropriate for a portfolio/development project but would need migration to a proper database for production use.

## Entity Relationships (Logical)

```
RESTAURANT (from zomato.csv)
    │
    ├── has many → CUISINE (multi-hot encoded)
    ├── has one  → LOCATION (Bangalore neighborhood)
    ├── has many → REVIEW (parsed from reviews_list)
    │                 └── has one → SENTIMENT (VADER score)
    ├── has one  → BAYESIAN_RATING (computed)
    ├── has one  → ETA_PREDICTION (from model, approximated)
    └── has one  → RELIABILITY_SCORE (computed)

DELIVERY_ORDER (from deliverytime.csv)
    │
    ├── has one  → RESTAURANT (via lat/long)
    ├── has one  → DELIVERY_LOCATION (lat/long)
    ├── has one  → DELIVERY_PERSON (age, rating, vehicle)
    ├── has one  → HAVERSINE_DISTANCE (computed)
    ├── has one  → TRAFFIC_CONDITION (ordinal)
    ├── has one  → FESTIVAL_STATUS (binary)
    ├── has one  → TIME_FEATURES (hour, day, weekend)
    └── has one  → ETA_PREDICTION (from model)
```

## Data Processing Rules

### Cleaning Rules

| Rule | Dataset | Justification |
|------|---------|---------------|
| Remove duplicates | Both | Identical rows provide no signal |
| Parse rating "X.X/5" → float | Zomato | Standardize to numeric |
| Parse cost "₹1,200" → number | Zomato | Standardize to numeric |
| Drop rows with missing rate | Zomato | Cannot impute restaurant ratings |
| Validate India lat/long (6-37, 68-98) | Delivery | Bangalore-specific bounds |
| Remove time > 120 min | Delivery | Physically improbable outliers |
| Fill categoricals with mode | Both | Conservative imputation |
| Fill numerics with median | Both | Robust to outliers |

### Feature Engineering Rules

| Feature | Formula | Purpose |
|---------|---------|---------|
| `votes_log` | log1p(votes) | Normalize skewed vote distribution |
| `haversine_distance_km` | Great-circle formula | Geographic distance metric |
| `traffic_ordinal` | Low=0, Medium=1, High=2, Jam=3 | Ordered traffic density |
| `bayesian_rating` | (v/(v+m))×R + (m/(v+m))×C | Prevent low-vote bias |
| `combined_score` | 0.5×similarity + 0.5×(bayesian/5) | Hybrid recommendation |
| `reliability_score` | 0.4×norm(rating) + 0.3×norm(sentiment) - 0.3×norm(delay) | Composite quality metric |
